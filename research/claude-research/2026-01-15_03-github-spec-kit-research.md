## Overview

**Repository**: GitHub Spec Kit (github/spec-kit)
**Purpose**: Spec-Driven Development (SDD) workflow framework
**Philosophy**: Specification before implementation

## Core Workflow

```
Constitution → Specify → Plan → Tasks → Implement
```

### Phase 1: Constitution
- Define project values and non-negotiables
- Establish architectural boundaries
- Set quality standards

### Phase 2: Specify
- Gather requirements through structured questions
- Document functional specifications
- Define acceptance criteria

### Phase 3: Plan
- Create implementation roadmap
- Identify dependencies
- Allocate resources

### Phase 4: Tasks
- Break down into atomic work units
- Assign priorities
- Define completion criteria

### Phase 5: Implement
- Execute tasks in priority order
- Validate against specifications
- Iterate if gaps found

## Key Principles

| Principle | Description |
|-----------|-------------|
| **Spec First** | No code without specification |
| **Atomic Tasks** | Small, focused work units |
| **Validation Gates** | Check before proceeding |
| **Iterative Refinement** | Specs evolve with understanding |
